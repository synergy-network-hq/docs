# Synergy Document Repository File List

This file is the root-level navigation guide for the organized Synergy document repository.

## Folder Map

- `01_Wallet_Product_And_Implementation/` — Wallet product requirements, platform behavior, UI/UX, account models, imports, and implementation specifications. Current file count: 162.
- `02_Network_Security_And_Cryptography/` — Security architecture, threat models, countermeasures, PQC/quantum-readiness, and verification requirements. Current file count: 148.
- `03_Standards_And_Identifiers/` — Canonical standards, Synergy identifiers, chain IDs, naming rules, address families, and protocol terminology. Current file count: 81.
- `04_Validator_Operations_And_Rewards/` — Validator, operator, uptime, rewards, hardware, and conduct standards. Current file count: 87.
- `05_Testnet_Operations_And_Incident_Reports/` — Testnet rollout, RPC/Atlas/indexer, validator recovery, incident reports, smoke tests, and operational evidence. Current file count: 82.
- `06_Token_Economics_And_Events/` — SNRG token event, utility, governance, pricing-stage, and token-economics materials. Current file count: 54.
- `99_Archive_And_Unclassified/` — Files that did not clearly classify elsewhere and should be reviewed manually. Current file count: 31.

## Naming and Version Rules

- Duplicate or near-duplicate files share the same canonical base name.
- Older-looking files are numbered with lower versions: `v0`, `v1`, `v2`, and so on.
- Newer-looking files receive higher version numbers. Version order uses filename dates, filename version markers, draft/final wording, and file modified time.
- Source filenames are preserved in `file_rename_manifest.csv`.
- Folder names are descriptive and must not be replaced with generic names like `Docs`, `Documents`, `Misc`, or `General`.
